# 종목 유사/조합 추천 MVP

국내(코스피/코스닥) 종목을 검색하면 같은 분야(업종) 유사 종목을 무료로 보여주고,
결제 후에는 상관계수 기반 "함께 오르는 종목(수혜주)"과 업종 방향성 요약을 제공하는 프로토타입입니다.

- 실제 서비스로 배포·수익화하는 전체 절차(컴퓨터 초보자용)는 [GUIDE.md](GUIDE.md)를 참고하세요.
- 현재까지의 개발 히스토리와 아키텍처 요약은 [HANDOFF.md](HANDOFF.md)를 참고하세요.

## 실행 방법

```bash
cd backend
venv/Scripts/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

브라우저에서 http://127.0.0.1:8000 접속.

최초 실행 시 KRX 전 종목 목록과 업종 정보를 자동으로 받아와 `backend/data/app.db` (SQLite)에 캐싱합니다.
가격 히스토리는 종목 조회 시점에 필요한 종목만 가져와 캐싱합니다 (20시간 이상 지나면 갱신).

## 결제(토스페이먼츠) 설정

결제는 **토스페이먼츠 결제위젯**을 사용합니다. 위젯 안에서 신용카드, 토스페이, 카카오페이, 페이코,
네이버페이 등을 사용자가 직접 선택할 수 있습니다 (모두 토스페이먼츠 한 번의 연동으로 제공됨).

기본값(키 미설정)으로 실행하면 토스페이먼츠 공식 문서의 **위젯 미리보기 전용 데모 키**로 결제 화면만
보여줍니다 — 화면 UI/UX 확인용이며 실제 결제 승인은 되지 않습니다.

**실제로 결제가 완료되도록 하려면** 토스페이먼츠 개발자센터에서 본인 테스트 키를 발급받아야 합니다.

1. https://developers.tosspayments.com 접속 후 이메일로 가입 (사업자 등록 불필요, 즉시 가입 가능)
2. 로그인 후 **개발자센터 → API 개별 연동 키** 메뉴에서 테스트 클라이언트 키(`test_ck_...`)와
   테스트 시크릿 키(`test_sk_...`)를 확인
3. 환경변수로 설정 후 서버 재시작:

```bash
export TOSS_CLIENT_KEY=test_ck_...
export TOSS_SECRET_KEY=test_sk_...
```

설정하면 결제하기 버튼을 눌러 실제로 테스트 결제가 승인 처리되고, 결과 페이지로 이동합니다.
테스트 모드에서는 임의의 카드번호(예: 4330-0000-0000-0000)와 유효기간·생년월일로 카드 결제를 완료할 수 있고,
토스페이·카카오페이 탭에서도 테스트 승인이 가능합니다.
가격은 `COMBO_PRICE_KRW` 환경변수로 조정 가능합니다 (기본 1,000원).

## 데이터 소스 / 로직

- 종목·업종: `FinanceDataReader`의 KRX-DESC 목록 (KSIC 업종 분류)
- 가격: `FinanceDataReader`의 일별 종가
- 유사 종목: 같은 업종(Industry) 내 시가총액 상위 종목
- 조합 추천("수혜주"): 같은 업종 내에서 최근 180일 일별 수익률의 피어슨 상관계수가 높은(함께 상승하는 경향이 강한) 종목 순, 양의 상관계수만 포함, 상위 2개
- 방향성: 업종 대표 종목들의 최근 20/60거래일 평균 등락률로 상승/횡보/하락 판정 (규칙 기반, LLM 미사용)

모든 추천/방향성 정보에는 투자 자문이 아니라는 면책 문구가 포함됩니다.

## 관리자/자동화 API

`ADMIN_TOKEN` 환경변수를 설정하면 아래 관리자 엔드포인트가 활성화됩니다 (미설정 시 503으로 비활성).
요청 헤더에 `X-Admin-Token: <ADMIN_TOKEN>`을 포함해야 합니다. 예약된 AI 에이전트(데이터 갱신/장애감시/SEO
콘텐츠 작성)가 이 엔드포인트들을 주기적으로 호출합니다.

- `POST /admin/refresh-data` — KRX 종목 데이터 강제 갱신
- `GET /admin/health` — 서버·데이터 상태 확인
- `POST /admin/posts` — "오늘의 시황" 글 등록 (`{"title": "...", "body": "..."}`)

공개 API: `GET /api/posts`, `GET /api/posts/{id}` — `backend/static/blog.html`, `blog-post.html`에서 사용.

## 법적 페이지

`backend/static/privacy.html`(개인정보처리방침), `terms.html`(이용약관), `about.html`(소개·문의) — 구글
애드센스 심사에 필요합니다. `about.html`의 이메일 주소는 실제 연락처로 교체해야 합니다.

## 알려진 제한사항 (MVP 범위)

- 실시간 시세가 아닌 일별 종가 기준
- Node.js가 설치되지 않은 환경 특성상 프론트엔드는 별도 빌드 없는 순수 HTML/JS(`backend/static`)로 구현
- 결제 세션은 로컬 SQLite에만 저장 (다중 서버 환경 미고려)
